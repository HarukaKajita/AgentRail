このスキルは、実装に進む前に必要情報が不足しているときに使う。

## いつ使うか

- `spec.md` の空欄禁止項目が埋められないとき
- `project.profile.yaml` の必須キーが不足しているとき
- 受入条件とテスト要件の対応が確定できないとき

## 入力

- 対象タスクID
- 未確定事項の一覧
- 現時点で確定している事実

## 出力

- `work/<task-id>/investigation.md` に「確認質問」セクションを追加・更新
- 優先度付き質問リスト（最大5件）
- `work/<task-id>/state.json` の `state` を `blocked` に設定

## 禁止事項

- 推測で仕様を確定しない
- 質問なしで `spec.md` を埋めない
- 無関係なファイルを変更しない
