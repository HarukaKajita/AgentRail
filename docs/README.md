# docs ディレクトリ運用ガイド

## 0. 前提知識 (Prerequisites) (必須)

- 参照資料:
  - `AGENTS.md`
  - `docs/INDEX.md`
- 理解ポイント:
  - 本資料に入る前に、目的・受入条件・依存関係を把握する。


`docs/` は SSOT (Single Source of Truth / 信頼できる唯一の情報源) を保持するドキュメントバンクです。  
仕様、意思決定、調査結果等の永続的な情報は、分散させず必ず本ディレクトリ配下で管理します。

## ディレクトリ構成

- `INDEX.md`: 全資料へのインデックス（導線管理）
- `templates/`: 各種ファイルのテンプレート
- `specs/`: 確定した要件仕様書
- `decisions/`: ADR (Architecture Decision Record / アーキテクチャ意思決定記録)
- `investigations/`: 永続的な調査・分析記録
- `operations/`: 運用手順書、ランブック、バックログ

## 運用ルール

1. 資料を新規追加・更新した際は、必ず `INDEX.md` を同時更新する。
2. 既存資料を変更した場合は、関連資料リンクの整合性を再確認する。
3. `work/<task-id>/` で確定した内容のみを `docs/` へ反映（永続化）する。
4. `templates/` 内のファイルを直接の作業用資料として使用しない。
5. 重要な設計・運用上の意思決定は、`decisions/` に ADR として記録する。
6. タスク完了前に `review.md` の `## 6. プロセス改善案 (Process Findings)` を確認し、重大度 `must/high` の項目は必ずフォローアップタスクへ接続する。

## 完了条件 (Done Criteria) との関係

以下のいずれかを満たさない変更は、タスク未完了とみなされます。

- 変更内容を反映したドキュメント (`docs/`) が適切に更新されていること。
- `INDEX.md` に追加・更新された資料への正しい導線が存在すること。
