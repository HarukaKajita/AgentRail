# 旧資料の隔離運用ポリシー

## 目的

現行運用の正本資料と、草案・履歴目的の旧資料を明確に分離する。

## 隔離対象

以下の資料は旧資料として `archive/legacy-documents/` に隔離し、通常の仕様判断では参照しない。

- `archive/legacy-documents/開発ワークフローと自動化.md`
- `archive/legacy-documents/フレームワーク設計と実装についてのChatGPTとの会話.md`

## 運用ルール

1. 現行の設計・実装判断は `docs/` と `work/` を正本として行う。
2. 隔離資料は履歴確認用途に限定し、通常の task 作業では参照不要とする。
3. 隔離資料を更新・復帰する場合は、明示的な依頼があるときのみ行う。
