# 開発段階用語集（旧称対応）

## 目的

過去資料に含まれる `Phase 1` / `Phase 2` の呼称を、現在の明示的な用語へ対応付ける。

## 用語対応

- 旧称: `Phase 1`
  - 新称: 手動運用導入段階
  - 意味: `work/<task-id>/` 成果物運用と docs 整備を中心に、手動プロセスを成立させる段階

- 旧称: `Phase 2`
  - 新称: 自動化基盤段階
  - 意味: `docs-indexer` / `consistency-check` / validator / CI 連携を導入して運用を自動化する段階

## 運用ルール

1. 新規資料では `Phase 1` / `Phase 2` を使わず、新称を使用する。
2. 既存 task-id に旧称が残る場合は履歴上の識別子として扱い、本文で新称を併記する。
3. 用語変更時は `docs/INDEX.md` と本資料を同時更新する。
