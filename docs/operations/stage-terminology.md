# 開発段階用語集

## 0. 前提知識 (Prerequisites) (必須)

- 参照資料:
  - `AGENTS.md`
  - `docs/INDEX.md`
- 理解ポイント:
  - 本資料に入る前に、目的・受入条件・依存関係を把握する。


## 目的

現行資料で使用する開発段階の名称を固定し、段階表現の曖昧さを防ぐ。

## 用語定義

- 手動運用導入段階
  - 意味: `work/<task-id>/` 成果物運用と docs 整備を中心に、手動プロセスを成立させる段階

- 自動化基盤段階
  - 意味: `docs-indexer` / `consistency-check` / validator / CI 連携を導入して運用を自動化する段階

## 運用ルール

1. 現行資料では本資料の正式名称のみ使用する。
2. 段階番号だけの略称や曖昧な表現は使用しない。
3. 用語変更時は `docs/INDEX.md` と本資料を同時更新する。
