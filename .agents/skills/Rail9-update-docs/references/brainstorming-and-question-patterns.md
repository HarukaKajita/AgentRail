# Brainstorming and Clarifying Question Patterns

この資料は、ユーザー要求から満足度の高い提案と確認質問を作るための共通ガイド。

## 1. 提案オプションの基本形

ユーザー要求を受けたら、少なくとも3案を提示する。

- Option A: 最短で目的達成（速度重視）
- Option B: バランス型（推奨）
- Option C: 将来拡張を優先（再作業を減らす）

各案で以下を必ず示す。

- 期待効果
- 実装コスト
- リスク
- このタスクへの適合理由

## 2. 確認質問の基本形

確認質問は 2〜4 件に制限し、影響の大きい順に並べる。

質問ごとに以下を明示する。

- 何を決める質問か
- その回答で何が変わるか

答えが曖昧でも進められる場合は、推奨案を仮定として明記する。
答えがないと進められない場合は、`blocked` 提案に切り替える。

## 3. 禁止事項

- 推測で仕様を断定しない。
- 意思決定に影響しない質問を増やさない。
- 提案なしで質問だけ投げない。

## 4. 最低出力フォーマット

1. 提案オプション（3案）
2. 推奨案と理由
3. 確認質問（2〜4件）
4. 次アクション（回答後の処理）
